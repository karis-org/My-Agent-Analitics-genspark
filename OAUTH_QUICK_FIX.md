# Google OAuth 404エラー - クイック修正ガイド

**所要時間**: 5分  
**最終更新**: 2025年11月02日

---

## 🚨 問題

Googleログインをクリックすると404エラーが発生する

---

## ✅ 解決方法（3ステップ）

### ステップ1: Google Cloud Consoleにアクセス

**直接リンク**: https://console.cloud.google.com/apis/credentials/consent

1. Googleアカウントでログイン
2. プロジェクト「webapp-439906」または「My Agent」を選択

---

### ステップ2: テストユーザーを追加（最重要！）

1. **「テストユーザー」タブ**をクリック
2. **「+ ADD USERS」**ボタンをクリック
3. **あなたのGmailアドレス**を入力
   ```
   例: realestate.navigator01@gmail.com
   ```
   
   ⚠️ **重要**: 
   - ✅ Gmailアドレスのみ使用可能
   - ❌ docomoメール（@docomo.ne.jp）は使用不可

4. **「追加」**ボタンをクリック
5. **「保存して次へ」**をクリック
6. **「ダッシュボードに戻る」**をクリック

---

### ステップ3: 5分待ってからテスト

1. ⏱️ **5-10分待つ**（Google側の設定反映に必要）
2. ブラウザのシークレットモードを開く
3. ログインページにアクセス:
   ```
   https://my-agent-analytics.pages.dev/auth/login
   ```
4. 「Googleでログイン」をクリック
5. テストユーザーとして追加したメールアドレスでログイン

---

## 🎯 期待される結果

✅ Googleアカウント選択画面が表示される  
✅ テストユーザーのメールアドレスが選択可能  
✅ ダッシュボードにリダイレクトされる

---

## 🐛 それでも404エラーが出る場合

### チェックリスト

- [ ] テストユーザーに**Gmailアドレス**を追加した（docomoメール不可）
- [ ] 設定を**保存**した
- [ ] **5-10分待った**
- [ ] **ブラウザキャッシュをクリア**した
- [ ] **シークレットモード**でテストした
- [ ] テストユーザーに追加したメールアドレスと**同じメールアドレス**でログインしようとしている
